# 스프링 MVC

- 스프링 MVC가 어떤 백그라운드를 가지고 있는지 알려면 서블릿부터 시작
- 서블릿은 HTTP 통신에 필요한 보일러플레이트를 제거해줌
    - 서버 TCP/IP 연결대기, 소켓 연결
    - HTTP 요청 메시지 파싱해서 읽기
    - ...
- 때문에 비즈니스 로직에만 집중 가능
- 하지만 이 서블릿에도 문제가 있음
    - `response.getWriter().write()` 이런식으로 해서 html을 자바코드에 다 넣어줘야함
    - 매우매우 불안한 구조
- 그리하여 JSP가 나옴
- 자바코드에 html을 넣는게 아니라 html을 기반으로 자바코드를 넣음
    - 때문에 동적으로 해줄부분만 자바코드삽입하여 유지보수가 비교적 쉬워짐
- 문제는 그런 일들을 해주기위해서 디비에서 데이터를 꺼내는 작업이나 비즈니스로직등이 함께 있어서 완전 html이랑 짬뽕 -> 유지보수 헬
- 이걸 해결하기 위해 MVC 패턴이라는 게 나옴
